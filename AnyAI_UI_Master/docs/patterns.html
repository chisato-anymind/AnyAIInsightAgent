パターン

実務で頻出する UI フローの標準構成。目的の体験を最短距離で。

一覧 / 詳細
  1. 検索・フィルタ・ソート（テーブルヘッダ/ツールバー）
  2. ページング（サーバー or 無限スクロール）
  3. 詳細ペインの 2 カラム化（>1024px）

サンプル: テーブル

作成 / 編集フォーム
  1. フィールドは 1 行 1 項目。ラベルと入力は縦積み。
  2. 検証メッセージはフィールド直下/赤色。
  3. アクションは右寄せ（保存 / キャンセル）。

サンプル: フォーム

  • ロード: スケルトン / スピナー
  • エラー: 問題 + 再試行 / サポート導線
